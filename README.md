# EAtool 胶粘 FPC 扣合固件 (EAtool_glue_FPCmating_Firmware)

本项目是基于 Arduino Uno 开发的 3 轴步进电机控制固件。主要用于 **EAtool 胶粘 FPC 压合** 设备的电机运动控制。固件实现了通过 **Modbus RTU** 协议与上位机（PLC/HMI/PC）进行串口通信，实现对三路步进电机的独立使能、速度设置、加速度设置、绝对/相对位置运动控制以及当前位置与状态的实时回传。

---

## 1. 硬件接口与引脚定义 (Pin Definitions)

固件运行在 **Arduino Uno** 主控板上，引脚配置如下：

### 1.1 步进电机引脚

| 电机编号 | 作用轴/描述 | 脉冲引脚 (Step Pin) | 方向引脚 (Dir Pin) | EEPROM 存储地址 | 默认脉冲当量 (steps/unit) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Motor 1** | 电机 1 (夹紧) | **D3** | **D4** | `0` (占用 0~3 字节) | `400.0` (如：400 步/mm 或度) |
| **Motor 2** | 电机 2 (放卷) | **D5** | **D6** | `4` (占用 4~7 字节) | `400.0` |
| **Motor 3** | 电机 3 (顶升) | **D7** | **D8** | `8` (占用 8~11 字节) | `400.0` |

> [!NOTE]
> **EEPROM 存储空间分配说明**：
> 在 Arduino Uno (AVR 架构) 中，单精度浮点数 `float` 占用 **4 字节**。为了防止电机之间掉电保存的位置数据相互覆盖，已将存储地址进行合理分配（电机 1 对应 `0`，电机 2 对应 `4`，电机 3 对应 `8`），各电机拥有独立的不重叠空间。

### 1.2 Modbus RTU 通信引脚

本项目使用 **SoftwareSerial** 模拟串口，并连接 **MAX485** 收发芯片。

| 引脚编号 | 引脚名称 | 连接设备管脚 | 描述 |
| :--- | :--- | :--- | :--- |
| **D10** | **RX** | MAX485 芯片 **RO (Receiver Output)** | 串口接收引脚 |
| **D11** | **TX** | MAX485 芯片 **DI (Driver Input)** | 串口发送引脚 |
| **D2** | **DE/RE** | MAX485 芯片 **DE & /RE** 并联 | 收发方向控制引脚（高电平发送，低电平接收） |
| **GND** | **GND** | MAX485 芯片 **GND** | 信号共地连接 |

## 2. 接线说明 (Wiring Instructions)

### 2.1 步进电机驱动器接线（共阴极接线）
项目硬件采用 **共阴极接线 (Common Cathode)** 方式，将步进电机驱动器（如 TB6600, DM542 等）与 Arduino Uno 连接：
1. **公共阴极 GND**：将驱动器的 **PUL- (PULSE-)**、**DIR- (DIRECTION-)** 和 **ENA- (ENABLE-)** 并联在一起，统一接到 Arduino 的 **GND** (共地)。
2. **脉冲控制信号**：将驱动器的 **PUL+ (PULSE+)** 接到 Arduino 对应的脉冲控制引脚上。
   - Motor 1 接 Arduino **D3**
   - Motor 2 接 Arduino **D5**
   - Motor 3 接 Arduino **D7**
3. **方向控制信号**：将驱动器的 **DIR+ (DIRECTION+)** 接到 Arduino 对应的方向控制引脚上。
   - Motor 1 接 Arduino **D4**
   - Motor 2 接 Arduino **D6**
   - Motor 3 接 Arduino **D8**
4. **使能控制信号**：将驱动器的 **ENA+ (ENABLE+)** 悬空（驱动器默认处于内部使能状态）。如果需要物理控制使能，可将其接至外部控制引脚。
5. **电机与电源**：
   - 将驱动器的 **VCC / GND** 接外部直流开关电源（一般选用 24V 至 48V DC，视电机规格而定）。
   - 将驱动器的 **A+、A-、B+、B-** 对应连接到 4 线双相步进电机的相线上。

### 2.2 Modbus RTU 通信接线 (使用 MAX485 芯片)
若使用独立的 MAX485 芯片或 MAX485 模块，接线方法如下：
1. **芯片 1 脚 (RO)** ─── Arduino **D10 (RX)**
2. **芯片 2 脚 (/RE) & 3 脚 (DE)** 并联 ─── Arduino **D2 (方向控制)**
3. **芯片 4 脚 (DI)** ─── Arduino **D11 (TX)**
4. **芯片 5 脚 (GND)** ─── Arduino **GND**
5. **芯片 8 脚 (VCC)** ─── Arduino **5V**
6. **芯片 6 脚 (A)** ─── 连接至 RS485 总线的 **A+** 信号线
7. **芯片 7 脚 (B)** ─── 连接至 RS485 总线的 **B-** 信号线

> [!TIP]
> 建议在 RS485 总线的末端（最远端的接收器 A 和 B 之间）并联一个 **120 欧姆** 的终端电阻，以消除高频信号反射，增强长距离通信的稳定性。

## 3. 工作原理与 Modbus 寄存器映射

### 3.1 核心工作原理
1. **主循环调度**：
   在 `loop()` 主循环中，系统首先运行 `getModbus().task()` 处理 Modbus 总线收发。随后，在 `handleMotorControl()` 中：
   - 从 Modbus 保持寄存器读取目标位置、最大运行速度和加速度数值。
   - 读取控制字，并解析控制指令。
   - 调用步进电机的底层执行函数 `run()`，由 `AccelStepper` 库基于高精度无阻塞的时间间隔判断生成运动脉冲。
   - 最后，在 `updateMotorStatus()` 中刷新电机状态字并写入当前的物理位置。
2. **高效能性能优化**：
   - **参数变化监测保护**：对目标位置、最大速度、最大加速度的设置增加了变化检测（Value-changed guard）。仅在参数值发生实际改变时，才会调用底层 `AccelStepper` 进行重新设定，避免了每个循环周期中重复执行高开销的 `sqrt` 和除法运算。这极大释放了 8-bit MCU 的 CPU 算力，使步进电机脉冲发生更加平滑稳定。
   - **控制状态机防抖**：优化了 `handleControlWord` 逻辑，在处于急停或未使能状态下，仅在状态切换的瞬间执行一次紧急停止及 EEPROM 保存，杜绝了禁用状态下的持续冗余比较与对 EEPROM.put（内部包含的只读比较）的高频高耗能调用。
3. **加减速控制**：
   `AccelStepper` 库会自动计算加减速阶段。在设定了加速度与最大速度后，电机会执行平滑的梯形速度曲线，能够防止高速启动时的失步和停止时的物理撞击。
4. **高可靠性边沿触发检测**：
   为避免上位机保持通信导致指令重复执行，固件对**运动触发 (Bit 1)** 与**初始化置零 (Bit 3)** 采用了**上升沿边沿触发检测**。指令只会且仅会在控制字中的相应位由 `0` 变为 `1` 的瞬间执行一次。此外，触发运动时立即将内部运行状态标记 `_runed` 置为 true，确保即便没有实质位移，电机依然能够产生运动完成 `STATUS_COMPLETED` 反馈并安全持久化位置。
5. **掉电位置持久化**：
   - 掉电恢复数据加载被移出全局静态构造函数，改由 `setup()` 调用 `loadPositionFromEEPROM()` 完成。这遵循了 Arduino 硬件与外设初始化的最佳安全实践，避免了在系统启动前读取 EEPROM 的潜在稳定性隐患。
   - 固件在以下场景会自动向 EEPROM 写入当前最新位置：
     - 电机运动正常结束，处于**目标点停止状态**时。
     - 触发**紧急停止 (Emergency Stop)** 锁定位置时。
     - 上位机触发**初始化置零 (Bit 3)** 操作成功时。
   - 由于只有在运动结束和重要事件发生时才进行单次写入，这极大地延长了 EEPROM 的写入寿命。

### 3.2 Modbus Holding Register (保持寄存器) 映射表

* **设备 Modbus 从站 ID (Slave ID)**: `1`
* **波特率 (Baudrate)**: `9600 bps` (数据位 8, 无校验，停止位 1)
* **起始寄存器地址**: `0` (所有电机寄存器均为 16-bit 长度)
* **每个电机占用寄存器数**: `10` 个

#### 电机寄存器整体分布
* **Motor 1** 寄存器范围：`0 ~ 9` (基准地址：`0`)
* **Motor 2** 寄存器范围：`10 ~ 19` (基准地址：`10`)
* **Motor 3** 寄存器范围：`20 ~ 29` (基准地址：`20`)

#### 单个电机寄存器偏移定义表 (以 Motor `i` 基准地址 `base = i * 10` 为例)

| 偏移地址 | 寄存器名 | 读写属性 | 数据类型 | 描述说明 |
| :---: | :--- | :---: | :--- | :--- |
| **+0** | **CONTROL_WORD** | R/W | `uint16_t` | **控制字**：包含启动、使能、初始化和急停控制，详见 3.3 节。 |
| **+1** | **STATUS_WORD** | R.O | `uint16_t` | **状态字**：展示电机使能、运行状态和错误标志，详见 3.4 节。 |
| **+2** | **TARGET_POS_H** | R/W | `float` (高 16 位) | **目标位置** (单位值)：与下一个寄存器拼成 32-bit Float 目标坐标。 |
| **+3** | **TARGET_POS_L** | R/W | `float` (低 16 位) | **目标位置** (单位值)：与前一个寄存器拼成 32-bit Float 目标坐标。 |
| **+4** | **SPEED_H** | R/W | `float` (高 16 位) | **最大速度** (单位值/s)：拼成 32-bit Float，电机最大限制速度。 |
| **+5** | **SPEED_L** | R/W | `float` (低 16 位) | **最大速度** (单位值/s)：拼成 32-bit Float，电机最大限制速度。 |
| **+6** | **ACCEL_H** | R/W | `float` (高 16 位) | **加速度** (单位值/s²)：拼成 32-bit Float，电机加减速的斜率。 |
| **+7** | **ACCEL_L** | R/W | `float` (低 16 位) | **加速度** (单位值/s²)：拼成 32-bit Float，电机加减速的斜率。 |
| **+8** | **CUR_POS_H** | R.O | `float` (高 16 位) | **当前位置** (单位值)：只读，拼成 32-bit 浮点数，实时刷新。 |
| **+9** | **CUR_POS_L** | R.O | `float` (低 16 位) | **当前位置** (单位值)：只读，拼成 32-bit 浮点数，实时刷新。 |

### 3.3 控制字 (CONTROL_WORD) Bit 位详细定义

控制字是 16 位的按位控制字段，通过向对应 Bit 写入 `1` 触发操作：

| Bit 位 | 对应十进制值 | 功能名称 | 描述与触发条件 |
| :---: | :---: | :--- | :--- |
| **Bit 0** | `0x0001` (1) | **使能 (Enable)** | 电机运行的**基础前提**。必须保持为 1 电机才能运动。如果此位写 0，电机会触发紧急制动并断开使能。 |
| **Bit 1** | `0x0002` (2) | **运动触发 (Run)** | **运动启动信号（上升沿触发）**。当该位由 `0` 变为 `1` 时，电机会读取目标位置并启动运动。运动过程中重复写 1 无效。 |
| **Bit 2** | `0x0004` (4) | **绝对/相对模式选择** | 选择运动的目标解析模式：<br>- **1**：绝对运动模式，运动至目标绝对坐标值 `_targetPos`。<br>- **0**：相对运动模式，向目标偏置移动相对距离 `_targetPos`。 |
| **Bit 3** | `0x0008` (8) | **置零/初始化 (Init)** | **初始化信号（上升沿触发）**。当该位由 `0` 变为 `1` 时，电机将当前的物理位置直接重置为 `0`，并同步持久化到 EEPROM。 |
| **Bit 4** | `0x0010` (16) | **急停 (Emergency Stop)** | **紧急停止信号**。写入 `1` 时，电机立即减速停止、锁定当前位置、清除运行标记，并强行将使能拉低。 |

---

### 3.4 状态字 (STATUS_WORD) Bit 位详细定义

状态字反映了电机当前的物理及逻辑状态：

| Bit 位 | 对应十进制值 | 状态名称 | 描述与反馈条件 |
| :---: | :---: | :--- | :--- |
| **Bit 0** | `0x0001` (1) | **已使能 (Enabled)** | 为 `1` 表示当前电机已使能上电，可以响应控制命令。 |
| **Bit 1** | `0x0002` (2) | **正在运行 (Running)** | 为 `1` 表示当前电机处于转动/加减速行进过程中。 |
| **Bit 2** | `0x0004` (4) | **运动完成 (Completed)** | 当电机执行过运动动作，并且正常到达目标位置停止时置 `1`。当有新的运动被触发时，该位自动清零。 |
| **Bit 3** | `0x0008` (8) | **错误标志 (Error)** | 电机驱动发生异常或故障时置 `1`。 |

### 3.5 代码接口设计与安全性防护（防参数颠倒设计）
为了提升代码健壮性并消除编译器的 Clang-Tidy 代码静态检查警告，固件对以下关键硬件参数和通信地址参数进行了类型封装：
- **`DeRePin` 结构体**：将原本为裸 `uint8_t` 的 MAX485 方向控制引脚（DE/RE）参数进行了结构体封装，并在构造函数中使用了 `explicit` 限定符：
  ```cpp
  struct DeRePin {
    uint8_t pin;
    explicit DeRePin(uint8_t p) : pin(p) {}
  };
  ```
  这避免了在调用 `setupModbus(deRePin, baudRate)` 时将引脚号与波特率（同为整型）传错顺序。
- **`ModbusAddr` 结构体**：将原本为裸 `int` 的 Modbus 寄存器起始地址参数封装为 `ModbusAddr` 结构体，并在其构造函数中添加 `explicit` 关键字。这防止了在调用 `writeFloatToModbus(value, addr)` 时，浮点型 `value` 与整型 `addr` 发生隐式转换或传错顺序的风险。

## 4. 典型控制流程示例

以控制 **Motor 1** 绝对运动到位置 **250.0** 为例，上位机操作步骤：

1. **设置运动参数**：
   - 将最大速度（例如 `400.0`）转换为 32-bit Float，写入寄存器 `4` (高字) 和 `5` (低字)。
   - 将加速度（例如 `800.0`）转换为 32-bit Float，写入寄存器 `6` (高字) 和 `7` (低字)。
2. **设置目标位置**：
   - 将目标位置 `250.0` 转换为 32-bit Float，写入寄存器 `2` (高字) 和 `3` (低字)。
3. **使能并触发运动**：
   - 组装控制字：
     - Bit 0 = 1 (使能)
     - Bit 1 = 1 (触发运动)
     - Bit 2 = 1 (绝对运动模式)
     - 控制字二进制表示为 `0000 0000 0000 0111`，十进制值为 `7` (`0x0007`)。
   - 向寄存器 `0` (CONTROL_WORD) 写入 `7`。因为是上升沿触发，电机 1 在收到该值的一瞬间启动运动。
4. **监控状态**：
   - 上位机循环读取寄存器 `1` (STATUS_WORD) 和寄存器 `8~9` (当前位置)。
   - 运动完成后，STATUS_WORD 中的 Bit 2 (对应值为 `4`) 将自动变为了 `1`。
   - 下次如果想再次运动，需要先将控制字 Bit 1 (Run) 写为 `0` 释放上升沿，接着重新设为 `1`。

## 5. 电机调试助手 (Motor Tuning Assistant)

为了方便测试，项目在 `tools/` 目录下提供了一个单文件、免安装的网页调试工具 **Motor Tuning Assistant.html**，支持通过 Web Serial API 直接在浏览器中连接设备并控制电机。

### 5.1 调试助手核心设计与安全校验
1. **无连接限制**：
   - 调试助手默认在未连接串口时禁用一切控制和写入按钮，并在用户点击时弹出友好警告（“请先连接串口”），防止误操作或因空指针导致程序报错。
2. **位置模式静默切换**：
   - 当用户在界面切换“绝对位置”或“相对位置”模式时，调试助手不会立即发送控制数据，而是仅在用户点击“运动触发 (Run)”按钮时，才会组合最新的控制字并一次性写入 Modbus 寄存器，从而避免总线充斥无效垃圾指令。
3. **高安全性连接握手校验**：
   - 为防止误连其他设备（例如连接到其他非本项目固件 of 串口设备导致误动作），调试助手在连接成功后，会自动触发包含 **首寄存器、尾寄存器、越界寄存器** 三步握手校验：
     - **步骤 1**：尝试读取 **第 0 号寄存器 (Motor 1 Control Word)**。若失败，判定连接无效。
     - **步骤 2**：尝试读取 **第 29 号寄存器 (Motor 3 Current Position Low)**。若失败，判定连接无效。
     - **步骤 3**：尝试读取 **第 30 号寄存器 (越界地址)**。此时固件必须返回 Modbus 异常码 `02` (Illegal Data Address)。若该地址能被正常读取，或者返回了非异常码，判定连错了其他设备并强制断开串口。
   - 只有三步校验全部通过后，调试助手才会正式解除界面锁定并开启周期为 1000ms 的自动轮询，以确保控制指令的绝对安全性。

## 6. 项目结构与编译说明

### 6.1 文件结构
```text
├── include/
│   ├── modbusReMap.h      # Modbus 寄存器映射与通信控制头文件

```

### 6.2 编译与上传
本项目使用 **PlatformIO** 进行开发。
1. 在 VS Code 中打开本项目文件夹。
2. PlatformIO 会自动检测 `platformio.ini` 并下载相关依赖库。
3. 点击 VS Code 底部状态栏的 **Build** (对勾图标) 进行编译。
4. 使用 USB 数据线连接 Arduino Uno 开发板，点击 **Upload** (右箭头图标) 进行固件烧录。

    *   引入了状态机的运行状态防抖保护。仅在电机从“使能”向“禁用/急停”过渡的瞬间调用 `emergencyStop()` 和执行一次位置持久化保存，从而避免了电机在已被禁用的状态下，每个 loop 周期的重复状态写入与对只读 EEPROM 的频繁访问。
    *   修正了原地或极短位移运动（目标差值为 0）无法完成触发状态反馈的隐患。在触发运动的上升沿瞬间立即将内部标志 `_runed` 置为 true，确保即便没有实质位移，电机依然能够产生运动完成 `STATUS_COMPLETED` 反馈并安全持久化位置。
*   **【优化】代码质量与静态警告修复**：
    *   将全局 ModbusRTU 实例改为通过 `getModbus()` 的方法获取引用，消除了头文件中的静态变量动态初始化风险警告，并同步修改了 `main.cpp` 中的调用。
    *   增加了 `ModbusAddr` 与 `DeRePin` 结构体并在构造函数上添加了 `explicit` 限定符，消消除了 `writeFloatToModbus` 和 `setupModbus` 中相邻同类型（或易隐式转换类型）参数易颠倒的警告。
    *   显式声明了 `MotorRegisterOffset` 枚举的底层数据类型为 `uint8_t`，优化内存占用。

### v1.1 (2026-05-22)
*   **【修复】EEPROM 存储地址重叠 Bug**：
    将三轴电机原先重叠的 EEPROM 地址 `0, 2, 4` 修改为 `0, 4, 8`（单精度浮点数占用 4 字节），解决因空间重叠导致的位置数据相互覆盖问题。
*   **【新增】EEPROM 掉电位置自动保存功能**：
    在代码中实现并补充了自动保存机制。当电机正常运行结束、触发急停、或是执行初始化置零时，固件会自动将最新位置持久化到 EEPROM，避免数据丢失。
*   **【新增】控制字按键上升沿边沿触发机制**：
    为控制字的“运动触发 (Bit 1)”与“初始化置零 (Bit 3)”引入上升沿检测逻辑。这解决了因上位机常送 1 信号而导致电机在相对运动或置零状态下无限循环触发的潜在重大隐患。
*   **【修复】C++ 类成员变量未初始化 Bug**：
    在 `Motor` 类的构造函数初始化列表中，显式赋初值给 `_runed`, `_completed`, `_running`, `_lastPos` 等变量，消除了未赋初值导致内存中存在垃圾值引起的未知错误。
*   **【优化】接线文档与上位机适配**：
    *   修正步进电机接线说明为正确的**共阴极接线**。
    *   细化了 **MAX485** 通信芯片的各个引脚连接细节与终端电阻配置说明。
