# Arduino EAtool 胶水/FPC压合控制固件 (Arduino_EAtool_glue_FPCmating_Firmware)

本项目是基于 Arduino Uno 开发的 3 轴步进电机控制固件。主要用于 **EAtool 胶水涂覆/FPC压合** 设备的电机运动控制。固件实现了通过 **Modbus RTU** 协议与上位机（PLC/HMI/PC）进行串口通信，实现对三路步进电机的独立使能、速度设置、加速度设置、绝对/相对位置运动控制以及当前位置与状态的实时回传。

---

## 1. 硬件接口与引脚定义 (Pin Definitions)

固件运行在 **Arduino Uno** 主控板上，引脚配置如下：

### 1.1 步进电机引脚

| 电机编号 | 作用轴/描述 | 脉冲引脚 (Step Pin) | 方向引脚 (Dir Pin) | EEPROM 存储地址 | 默认脉冲当量 (steps/unit) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Motor 1** | 电机 1 (如X轴/点胶) | **D3** | **D4** | `0` | `1600.0` (如：1600步/mm 或度) |
| **Motor 2** | 电机 2 (如Y轴/压合) | **D5** | **D6** | `2` *(有重叠风险, 见注意事项)* | `400.0` |
| **Motor 3** | 电机 3 (如Z轴/升降) | **D7** | **D8** | `4` *(有重叠风险, 见注意事项)* | `1600.0` |

> [!WARNING]
> **EEPROM 地址重叠风险**：在 Arduino Uno (AVR) 架构中，单精度浮点数 `float` 占用 **4 字节**。当前代码中 Motor 1、2、3 对应的 EEPROM 起始地址分别为 `0, 2, 4`。
> - Motor 1 占用地址 `0 ~ 3`
> - Motor 2 占用地址 `2 ~ 5` （与 Motor 1 重叠）
> - Motor 3 占用地址 `4 ~ 7` （与 Motor 2 重叠）
>
> **建议修改**：为避免掉电保存时数据相互覆盖，建议将存储地址修改为不重叠的间隔（例如：Motor 1 = `0`，Motor 2 = `4`，Motor 3 = `8`）。

### 1.2 Modbus RTU 通信引脚

本项目使用 **SoftwareSerial** 模拟串口，并连接 RS485 转换模块。

| 引脚编号 | 引脚名称 | 连接设备管脚 | 描述 |
| :--- | :--- | :--- | :--- |
| **D10** | **RX** | RS485 模块 **TXD (RO)** | 串口接收引脚 |
| **D11** | **TX** | RS485 模块 **RXD (DI)** | 串口发送引脚 |
| **D2** | **DE/RE** | RS485 模块 **DE / RE** 并联 | 收发方向控制引脚（高电平发送，低电平接收） |
| **GND** | **GND** | RS485 模块 **GND** | 共地连接 |

## 2. 接线说明 (Wiring Instructions)

### 2.1 步进电机驱动器接线（以共阳极接线为例）
以单轴为例，将步进电机驱动器（如 TB6600, DM542 等）与 Arduino Uno 连接：
1. **驱动器 PUL+ (PULSE+)** / **DIR+ (DIRECTION+)** / **ENA+ (ENABLE+)**：统一并联接 Arduino 的 **+5V**。
2. **驱动器 PUL- (PULSE-)**：接对应的 Arduino 脉冲引脚（如 Motor 1 接 **D3**）。
3. **驱动器 DIR- (DIRECTION-)**：接对应的 Arduino 方向引脚（如 Motor 1 接 **D4**）。
4. **驱动器 ENA- (ENABLE-)**：可悬空（默认使能），或接 Arduino 的 IO 口以实现物理使能控制（当前固件主要通过控制字软件使能/停止）。
5. **驱动器电源 (VCC/GND)**：接外部步进电机电源（通常为 24V - 48V DC）。
6. **驱动器 A+/A-/B+/B-**：接步进电机对应的相线。

### 2.2 RS485 通信模块接线
使用 MAX485 或同类型 TTL 转 RS485 模块：
1. **VCC** ─── Arduino **5V**
2. **GND** ─── Arduino **GND**
3. **RXD (RO)** ─── Arduino **D10**
4. **TXD (DI)** ─── Arduino **D11**
5. **DE / RE** (并联短接) ─── Arduino **D2**
6. **A+ / B-** ─── 连接上位机/主站的 **RS485 A / B** 物理总线

## 3. 工作原理与 Modbus 寄存器映射

### 3.1 核心工作原理
1. **主循环机制**：
   在 `loop()` 中，系统通过 `mb.task()` 轮询 Modbus 通信；通过 `handleMotorControl()` 处理来自寄存器的控制字与目标参数，驱动电机执行步进，并调用 `m->run()` 刷新步进电机的脉冲输出；通过 `updateMotorStatus()` 实时刷新电机的状态字与当前位置。
2. **运动引擎**：
   底层使用 `AccelStepper` 库的 `DRIVER` 模式。该模式会自动根据设置的 **最大速度** 与 **加速度** 计算机身梯形速度曲线，实现平滑的启动和停止。
3. **掉电位置保持**：
   在系统启动 `setup()` 时，电机会尝试从各自的 EEPROM 地址读取上次保存的当前位置。若读取到有效值，则将其设为当前位置，从而实现掉电位置恢复。

### 3.2 Modbus Holding Register (保持寄存器) 映射表

* **设备 Modbus 从站 ID (Slave ID)**: `1`
* **波特率 (Baudrate)**: `9600 bps` (数据位 8, 无校验, 停止位 1)
* **起始寄存器地址**: `0`
* **每个电机占用寄存器数**: `10` 个 (每个寄存器为 16-bit)

#### 电机寄存器整体分布
* **Motor 1** 寄存器范围: `0 ~ 9`
* **Motor 2** 寄存器范围: `10 ~ 19`
* **Motor 3** 寄存器范围: `20 ~ 29`

#### 单个电机寄存器偏移定义表 (以 Motor 1 基准地址 `0` 为例)

| 偏移地址 | 寄存器名 | 读写属性 | 数据类型 | 描述说明 |
| :---: | :--- | :---: | :--- | :--- |
| **+0** | **CONTROL_WORD** | R/W | `uint16_t` | **控制字**：写入不同 Bit 触发电机动作，详见 3.3 节。 |
| **+1** | **STATUS_WORD** | R.O | `uint16_t` | **状态字**：指示电机当前运行状态，详见 3.4 节。 |
| **+2** | **TARGET_POS_H** | R/W | `float` (高16位) | **目标位置** (单位值)：与下一个寄存器拼成 32-bit Float。 |
| **+3** | **TARGET_POS_L** | R/W | `float` (低16位) | **目标位置** (单位值)：与前一个寄存器拼成 32-bit Float。 |
| **+4** | **SPEED_H** | R/W | `float` (高16位) | **最大运行速度** (单位值/s)：拼成 32-bit Float。 |
| **+5** | **SPEED_L** | R/W | `float` (低16位) | **最大运行速度** (单位值/s)：拼成 32-bit Float。 |
| **+6** | **ACCEL_H** | R/W | `float` (高16位) | **加速度** (单位值/s²)：拼成 32-bit Float。 |
| **+7** | **ACCEL_L** | R/W | `float` (低16位) | **加速度** (单位值/s²)：拼成 32-bit Float。 |
| **+8** | **CUR_POS_H** | R.O | `float` (高16位) | **当前实际位置** (单位值)：只读，拼成 32-bit Float。 |
| **+9** | **CUR_POS_L** | R.O | `float` (低16位) | **当前实际位置** (单位值)：只读，拼成 32-bit Float。 |

> [!NOTE]
> **浮点数 (Float) 拼接规则**：
> - 固件中采用大端传输（即高16位寄存器在前，低16位寄存器在后）。
> - 上位机在读取或写入位置/速度/加速度时，需要将两个连续的 16 位寄存器组合为一个标准的 32 位 IEEE 754 浮点数。
> - 例如：Motor 1 的目标位置在高字寄存器 `2` 和低字寄存器 `3` 中。

### 3.3 控制字 (CONTROL_WORD) Bit 位定义

控制字是 16 位的按位控制字段，通过向对应 Bit 写入 `1` 触发操作：

| Bit 位 | 十进制值 | 功能名称 | 描述说明 |
| :---: | :---: | :--- | :--- |
| **Bit 0** | `0x0001` | **使能 (Enable)** | **必须保持为 1 电机才能运动**。若此位为 0，则触发紧急停止并取消使能状态。 |
| **Bit 1** | `0x0002` | **运动触发 (Run)** | **上升沿/触发信号**。当此位为 1 且电机未在运行时，启动运动，向目标位置移动。 |
| **Bit 2** | `0x0004` | **绝对/相对位置选择** | - **1**：绝对运动（走至目标绝对坐标 `moveTo(target)`）<br>- **0**：相对运动（相对当前位置运动 `move(target)`） |
| **Bit 4** | `0x0010` | **急停 (Emergency Stop)** | 写入 **1** 触发急停，电机立即减速停止并锁死当前位置，取消使能。 |

*注：其他未定义的 Bit 位预留，请保持为 0。*

---

### 3.4 状态字 (STATUS_WORD) Bit 位定义

状态字反映了电机当前的物理及逻辑状态：

| Bit 位 | 十进制值 | 状态名称 | 描述说明 |
| :---: | :---: | :--- | :--- |
| **Bit 0** | `0x0001` | **已使能 (Enabled)** | 指示当前电机是否处于正常使能工作状态。 |
| **Bit 1** | `0x0002` | **正在运行 (Running)** | 指示当前电机是否正在转动/运动中。 |
| **Bit 2** | `0x0004` | **运动完成 (Completed)** | 运动结束且到达目标位置时置 1。新运动启动时自动清零。 |
| **Bit 3** | `0x0008` | **错误标志 (Error)** | 电机出现异常或故障时置 1。 |

## 4. 典型控制流程示例

以控制 **Motor 1** 绝对运动到位置 **100.0** 为例，上位机操作步骤：

1. **设置运动参数**：
   - 将最大速度（例如 `200.0`）转换为 32-bit Float，写入寄存器 `4` (高字) 和 `5` (低字)。
   - 将加速度（例如 `500.0`）转换为 32-bit Float，写入寄存器 `6` (高字) 和 `7` (低字)。
2. **设置目标位置**：
   - 将目标位置 `100.0` 转换为 32-bit Float，写入寄存器 `2` (高字) 和 `3` (低字)。
3. **使能并触发运动**：
   - 组装控制字：
     - Bit 0 = 1 (使能)
     - Bit 1 = 1 (触发运动)
     - Bit 2 = 1 (绝对运动模式)
     - 控制字十进制值为 `7` (`0x0007`)。
   - 向寄存器 `0` (CONTROL_WORD) 写入 `7`。
4. **监控状态**：
   - 上位机循环读取寄存器 `1` (STATUS_WORD) 和寄存器 `8~9` (当前位置)。
   - 当读取到状态字中的 Bit 2 (值 `0x0004`) 变为 1 时，表示运动已成功到达 `100.0`。
   - 下次需要再次运动时，需要将控制字 Bit 1 重新置零后再置一（上升沿触发）。

## 5. 项目结构与编译说明

### 5.1 文件结构
```text
├── include/
│   ├── modbusReMap.h      # Modbus 寄存器映射与通信控制头文件
│   └── motor.h            # 电机控制类封装定义（继承自 AccelStepper）
├── src/
│   ├── main.cpp           # 程序入口、硬件初始化与主循环逻辑
│   ├── modbusReMap.cpp    # Modbus 数据收发及寄存器读写实现
│   └── motor.cpp          # 电机参数设置、控制字解析、EEPROM 加载实现
├── platformio.ini         # PlatformIO 项目配置文件 (定义了Uno开发板和依赖库)
└── Mbpoll1.mbp            # Modbus Poll 测试软件的预设工程配置文件
```

### 5.2 编译与上传
本项目使用 **PlatformIO** 进行开发。
1. 安装 VS Code，并在扩展市场中搜索并安装 **PlatformIO IDE**。
2. 在 VS Code 中打开本项目文件夹。
3. PlatformIO 会自动检测 `platformio.ini` 并下载相关依赖库（`AccelStepper` 和 `modbus-esp8266`）。
4. 点击 VS Code 底部状态栏的 **Build** (对勾图标) 进行编译。
5. 使用 USB 数据线连接 Arduino Uno 开发板，点击 **Upload** (右箭头图标) 进行固件烧录。
